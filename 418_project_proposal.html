Summary

We are going to implement a parallel form of a minimax algorithm that uses alpha-beta cutoffs for an AI that plays Othello.

Background

The minimax algorithm is a decision making algorithm, very commonly used in game AIs, to determine the best possible move one player could make. Each of the children of the nodes is all of the possible moves that could follow, and tracks the “score” of that turn, and whether as a whole that branch of moves would be more beneficial or harmful in the long term for the game.

Alpha Beta pruning is a method that makes the calculation of this tree faster, as it would obviously increase exponentially. So if a branch will always have a lower score than the best guaranteed score, than the algorithm stops calculating that branch, reducing the size of the tree.

Challenge

The initial thought to parallelize a minimax tree is to assign each of the different initial moves to different threads. However, the issue with this would be that different threads would not have updated information on alpha-beta cutoff values for other threads. The challenge in this project would be implementing a different way of distributing the work, or a clever way of maintaining the cutoff values between threads, or a combination of the two. The workload is compute intensive, and does not require much memory transfer.

Resources

We plan on working on the GHC machines. We will be starting from scratch rather than an existing codebase. Currently, we are using the paper "Efficiency of Parallel Minimax Algorithm for Game Tree Search" by Plamenka Borovska and Milena Lazarova, for an initial implementation and will try to build off of this to see the effects of various strategies.

Goals and Deliverables

Plan to Achieve

  • 4x speedup utilizing the 4 cores of the GHC machines
  • Comparing the speedup of different alpha beta pruning techniques

Hope to Achieve

  • Front end web app

Platform Choice

We will be writing in C++ and testing on the GHC machines. We will be using threads to implement parallelization because the work of calculating the minmax trees different branches is suited to assigning it to different threads.

Another reason C++ makes sense is because then we can convert our code into a shared object and call it in a python web server.
